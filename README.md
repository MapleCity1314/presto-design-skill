# Presto Design

`presto-design` is a modular frontend design skill for Codex-style agents. It routes UI requests into focused clusters for create, refine, redesign, motion, fix, and overdrive work, then applies shared contract rules and optional aesthetic presets.

## What this repo contains

- [`presto-design/SKILL.md`](./presto-design/SKILL.md): the skill entrypoint and routing logic
- [`presto-design/clusters/`](./presto-design/clusters): task-specific workflows
- [`presto-design/presets/`](./presto-design/presets): aesthetic presets
- [`presto-design/contract.md`](./presto-design/contract.md): cross-cluster output rules
- [`presto-design/evals/`](./presto-design/evals): eval definitions, judge prompt, and generated results
- [`docs/`](./docs): design notes and planning artifacts
- [`tests/`](./tests): local fixtures and demo workspaces used during iteration

## Skill behavior

The skill acts as a router:

- `create`: build UI from scratch
- `refine`: improve specific elements without changing the whole composition
- `redesign`: upgrade an existing project in place
- `motion`: add purposeful motion
- `fix`: audit and correct quality issues
- `overdrive`: push the interface toward a more ambitious result

It can also apply presets such as `minimalist` and `editorial`, while enforcing a shared anti-slop contract.

## Local development

1. Edit files inside [`presto-design/`](./presto-design).
2. Use fixtures in [`tests/`](./tests) for manual runs and eval work.
3. Keep the skill body lean and move detailed logic into cluster files or references.
4. Re-run evals after meaningful routing or contract changes.
